Photo Manager

mauriciocastellanos.com — gallery editor
Open the site-preview folder to get started.

No folder open

Click Open site folder… above and select your site-preview folder (the one containing index.html and photos-data.js). This tool only works in Chrome or Edge.

Changes are written straight to photos-data.js and the Pictures folder on your computer — nothing is uploaded anywhere. Re-open your local site (or redeploy) afterwards to see the changes.

Deploy

Create a token at github.com → Settings → Developer settings → Personal access tokens → Fine-grained, scoped to just this repo with Contents: Read and write permission. It's stored only in this browser (localStorage) and used solely to call GitHub's API directly from here — nothing else ever sees it.

Add photos

Click to choose image files, or drag & drop them here.

Each photo needs a caption (short title shown on hover) and a description (used as alt text — good for accessibility and SEO). Pick an existing gallery or create a new one. Files are copied into Pictures/<Gallery>/ with a cleaned-up filename when you save.

Galleries

Every gallery here becomes a filter tab on the site, in this order (after "All"). Deleting a gallery is only allowed once it has no photos left.

Existing photos

Use Rename file on any photo to clean up a messy filename — it renames the file on disk and updates its path. Don't forget to click Save changes afterwards.
